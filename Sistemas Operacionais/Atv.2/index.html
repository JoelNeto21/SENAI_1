1. Definindo os Paradigmas: Código Aberto vs. Fechado

Tarefa: Descreva com suas palavras o que é um sistema operacional de código aberto e um sistema operacional de código fechado. Cite ao menos duas vantagens e duas desvantagens de cada modelo. Apresente três exemplos de cada tipo.

Um sistema operacional de código aberto é aquele cujo código-fonte é disponibilizado ao público, permitindo modificações e redistribuição. Já o sistema de código fechado tem seu código protegido, sendo controlado exclusivamente pela empresa desenvolvedora.

Vantagens do código aberto: Gratuito ou de baixo custo Alta personalização.
Desvantagens: Pode exigir mais conhecimento técnico; Suporte técnico pode depender da comunidade.

Vantagens do código fechado: Interface amigável para usuários leigos; Suporte oficial garantido.
Desvantagens: Licenças geralmente são pagas; Pouca flexibilidade para modificações.

Exemplos de código aberto: Linux (Ubuntu, Fedora, Debian).
Exemplos de código fechado: Windows, macOS, iOS.

2. Análise Comparativa de Instalação

Tarefa: Pesquise e descreva as principais diferenças que um usuário encontraria durante o processo de instalação de um sistema operacional de código fechado (ex: Windows) em comparação com um de código aberto (ex: Ubuntu). Considere aspectos como licenciamento, opções de personalização durante a instalação e necessidade de drivers.

A instalação do Windows requer a aceitação de uma licença paga, enquanto o Ubuntu é gratuito. Durante a instalação, o Ubuntu oferece mais opções de personalização (como escolha de pacotes e particionamento avançado). O Windows tem suporte mais automatizado a drivers, enquanto o Ubuntu pode precisar de configuração manual ou instalação posterior de drivers proprietários.

3. Debate: Modo Texto vs. Interface Gráfica

Tarefa: Elabore um texto argumentativo discutindo as vantagens e desvantagens do uso do modo texto (linha de comando) em relação à interface gráfica (GUI) para realizar tarefas em um sistema operacional. Em quais cenários cada um é mais apropriado?

O modo texto oferece maior controle e automação de tarefas, sendo ideal para administradores de sistemas e servidores. Ele consome menos recursos e permite executar tarefas rapidamente via scripts. Porém, exige mais conhecimento técnico. Já a interface gráfica é intuitiva e ideal para usuários comuns, facilitando a navegação e o uso de recursos. Em servidores e ambientes profissionais, o modo texto é preferido; em desktops, a GUI é mais prática.

4. Estruturas de Diretórios: Uma Análise Comparativa

Tarefa: Pesquise e compare a estrutura de diretórios padrão de um sistema operacional Windows (ex: C:\Windows, C:\Program Files, C:\Users) com a de um sistema Linux (ex: /bin, /etc, /home, /var). Explique a finalidade de pelo menos três diretórios principais de cada sistema.

No Windows:
- C:\Windows: contém os arquivos do sistema operacional.
- C:\Program Files: armazena programas instalados.
- C:\Users: guarda os dados dos perfis de usuários.

No Linux:
- /bin: contém comandos essenciais do sistema.
- /etc: possui arquivos de configuração.
- /home: guarda os diretórios pessoais dos usuários.

5. Modelos de Usuários e Permissões

Tarefa: Explique o conceito de gerenciamento de usuários e permissões em sistemas operacionais. Compare como as permissões de arquivos e diretórios são tipicamente gerenciadas em um sistema de código fechado (como o Windows, com NTFS) e em um de código aberto (como o Linux, com o sistema rwx).

O gerenciamento de usuários e permissões define quem pode acessar, modificar ou executar arquivos. No Windows (NTFS), as permissões são gerenciadas por ACLs (Listas de Controle de Acesso), com uma interface gráfica. No Linux, as permissões seguem o padrão rwx (leitura, escrita e execução) para usuário, grupo e outros, sendo gerenciadas via terminal com comandos como chmod e chown.

6. O Processo de Boot: Do Power-On ao Desktop

Tarefa: Descreve as etapas gerais do processo de inicialização (boot) de um computador, desde o momento em que é ligado até o carregamento completo do sistema operacional e a apresentação da interface ao usuário. Destaque se há diferenças significativas nesse processo entre sistemas de código aberto e fechado.

O processo de boot inicia com o POST (autoteste de hardware), depois o BIOS/UEFI busca um dispositivo de boot. O carregador de boot (como GRUB no Linux ou o Boot Manager do Windows) é chamado e inicia o kernel do sistema operacional. Em seguida, são carregados os drivers e serviços essenciais, até chegar à interface gráfica. Sistemas abertos como o Linux costumam permitir mais personalizações e mostram mais detalhes técnicos durante o boot.

7. Redes: Compartilhamento, Mapeamento, deve ser criado um exemplo.

Tarefa: Defina:
a) O que são unidades e pastas em rede e qual sua utilidade.
b) O que significa "compartilhamento" e "acesso" no contexto de recursos de rede.
c) O que é mapeamento de unidades de rede.

a) Unidades e pastas em rede são recursos armazenados em outro computador ou servidor e acessados por múltiplos usuários na mesma rede. Servem para centralizar arquivos e facilitar o trabalho colaborativo.
b) Compartilhamento é tornar uma pasta ou recurso disponível a outros usuários da rede; acesso refere-se às permissões concedidas (ler, escrever, etc.).
c) Mapeamento é atribuir uma letra (como Z:) a uma pasta de rede para facilitar o acesso como se fosse um disco local.

8. Fundamentos de Segurança Cibernética

Tarefa:
a) Defina Segurança Cibernética.
b) Liste e descreva brevemente 5 tipos comuns de ameaças cibernéticas.
c) Explique o que são vulnerabilidades em um sistema.
d) O que são "credenciais" e por que sua proteção é crucial?
e) Defina Engenharia Social e dê dois exemplos.

a) Segurança Cibernética é o conjunto de práticas, tecnologias e processos usados para proteger sistemas, redes e dados contra ataques digitais.

b) Tipos comuns de ameaças:
- Malware – software malicioso como vírus e trojans;
- Phishing – tentativa de roubo de dados via e-mails falsos;
- Ransomware – sequestro de dados com pedido de resgate;
- Ataques DDoS – sobrecarregam servidores;
- Spyware – coleta dados do usuário secretamente.

c) Vulnerabilidades são falhas ou brechas em sistemas que podem ser exploradas por atacantes.

d) Credenciais são os dados de autenticação (usuário e senha). Protegê-las é essencial para evitar acessos indevidos.

e) Engenharia Social é a manipulação de pessoas para obter informações ou acesso. Exemplos:
- Fingir ser técnico para pedir senhas;
- Enviar e-mails falsos com links maliciosos.

9. Fundamentos de Segurança Cibernética

Tarefa: Discuta quais são as principais intervenções, métodos de proteção e prevenção que podem ser adotados por usuários e administradores para aumentar a segurança de um sistema operacional e dos dados. Explique a função de um firewall nativo de sistema operacional neste contexto.

Entre os métodos estão: manter o sistema e softwares atualizados, usar senhas fortes, ativar autenticação em duas etapas, instalar antivírus, e configurar corretamente permissões de acesso. O firewall nativo (como o do Windows) atua monitorando e filtrando o tráfego de rede, bloqueando conexões suspeitas e impedindo acessos não autorizados ao sistema.

10. Configuração do Sistema: Pontos Críticos

Tarefa: Identifique e descreva pelo menos cinco áreas de configuração críticas em um sistema operacional (tanto aberto quanto fechado) que um administrador ou usuário avançado precisaria conhecer para otimizar o desempenho, a segurança e a usabilidade do sistema.

- Gerenciador de inicialização: controla os programas que iniciam com o sistema, impactando o desempenho.
- Atualizações automáticas: garantir que o sistema receba correções de segurança.
- Permissões e contas de usuário: controlar quem pode acessar e modificar arquivos.
- Configurações de rede: IP, DNS, firewall e compartilhamentos.
- Gerenciamento de energia: especialmente importante em notebooks para otimizar bateria e desempenho.
