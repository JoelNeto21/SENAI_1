Situação de Aprendizagem - Atividade Individual

Ameaças Cibernéticas Cenário de Phishing:

Um funcionário recebe um e-mail aparentemente legítimo de seu banco, solicitando que ele clique em um link para verificar informações da conta. O link leva a uma página falsa que coleta suas credenciais. Demonstrar como os ataques de phishing funcionam e como identificar sinais de um e-mail falso.

O phishing é um ataque cibernético muitos comum, normalmente com emails(spam) e SMS's falsos. Onde o usuário, ao cair no golpe, acaba passando informações confidenciais ao golpista. Para evitar tais situações, é recomendado que sempre confirme a origem das mensagens, e nunca forneça suas informações ̶ empresas raramente pedem essas informações por esses canais.

Cenário de Ransomware:

Um usuário baixa um arquivo de um site desconhecido. O arquivo contém ransomware que criptografa todos os dados do computador e exige um pagamento para liberar os arquivos. Objetivo: Mostrar os riscos de baixar arquivos de fontes não confiáveis e a importância de backups regulares.

Jamais faça um dowload vindo de um site “suspeito” da internet. Ao fazer isso você acaba colocando seu computador e suas informações em risco. Também é recomendável, que se faça backups regularmente do sistema, garantindo que não haja perda de informações importantes.

Cenário de Senhas Fracas:

Um sistema tem várias contas de usuário com senhas simples como "123456" e "password". Um atacante usa um ataque de força bruta para acessar essas contas.

Senhas fracas são brechas enormes que deixamos para possíveis malfeitores digitais. O ideial é sempre utilizarmos senhas mais robustas (letras maiúsculas, minúsculas, números, caracteres especiais...) dificultando a vida dos atacantes ̶ sim, “dificultando”, porque mesmo assim há chances deles burlarem o sistema de segurança e acessar seus dados, só nos resta confiar na criptografia. Usar gerenciadores de senhas é uma ótima opção.

Cenário de Software Desatualizado:

Um servidor está executando uma versão antiga de um software que possui vulnerabilidades conhecidas. Um atacante explora essas vulnerabilidades para obter acesso ao servidor. Destacar a importância de manter o software atualizado para evitar explorações.

Todos estamos em constante evolução. E porque seria diferente para os softwares? Atualizações periódicas são de extrema importâcia para mantermos um sistema seguro ̶ eliminando possíveis brechas de segurança ̶ e livre de bugs.
